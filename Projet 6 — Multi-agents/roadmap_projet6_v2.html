01
🏗️
Créer les 4 sous-workflows spécialisés
Execute Workflow Trigger · chacun fait une seule chose
NOUVEAU
▶

Crée 4 nouveaux workflows dans N8N. Chaque workflow démarre avec un Execute Workflow Trigger — c'est lui qui permet à l'agent chef de les appeler comme des fonctions.

Sous-agent 1 — "Recherche Leads" Trigger : Execute Workflow Trigger Action : Apify → GPT scoring → retourne les leads Sous-agent 2 — "Rédaction Email" Trigger : Execute Workflow Trigger Action : GPT copywriting → retourne l'email rédigé Sous-agent 3 — "Analyse Données" Trigger : Execute Workflow Trigger Action : Code node stats → retourne le rapport Sous-agent 4 — "Actions CRM" Trigger : Execute Workflow Trigger Action : Airtable + Notion + Gmail
02
🧠
Créer l'agent chef orchestrateur
AI Agent + Call n8n Workflow Tool × 4
NOUVEAU !
▶

L'agent chef utilise le node AI Agent avec 4 tools de type Call n8n Workflow. Il reçoit une demande, analyse et appelle le bon sous-workflow automatiquement.

System prompt : Tu es un orchestrateur IA expert. Tu délègues au bon sous-agent selon la demande. Règles : - Recherche leads/contacts → sous-agent-recherche - Rédiger email/post/texte → sous-agent-redaction - Analyser données/stats → sous-agent-analyse - Actions CRM/Calendar → sous-agent-actions Retourne toujours : { "action": "...", "resultat": "...", "statut": "ok/erreur" }
03
🔗
Connecter via Call n8n Workflow Tool
le node clé de ce projet
CLÉ
▶

Dans l'agent chef, chaque tool est un Call n8n Workflow Tool qui pointe vers un sous-workflow. C'est le nouveau node de ce projet !

Tool 1 : "sous-agent-recherche" → Workflow : Sous-agent Recherche Leads → Input : { "cible": "agences marketing Paris", "limit": 5 } → Output : { "leads": [...] } Tool 2 : "sous-agent-redaction" → Workflow : Sous-agent Rédaction Email → Input : { "entreprise": "...", "contexte": "..." } → Output : { "sujet": "...", "corps": "..." }
04
⚠️
Gestion d'erreurs — Error Trigger + Retry
ce qui différencie un junior d'un senior
AVANCÉ
▶

Pour chaque sous-workflow, on ajoute un Error Trigger qui se déclenche si le workflow plante. Il log l'erreur, envoie une alerte Slack et relance automatiquement jusqu'à 3 fois.

Error Trigger → se déclenche si le workflow plante ↓ Code node → extrait le message d'erreur ↓ IF retry_count < 3 ├── True → Wait 30s → Relance le workflow └── False → Log Google Sheets + Alerte Slack 🚨
05
🧪
Test complet — mission de prospection réelle
du début à la fin sans intervention humaine
TEST FINAL
▶

On donne à l'agent chef cette mission et on regarde s'il orchestre tout seul :

"Trouve 3 agences marketing à Paris, rédige un email de prospection personnalisé pour chacune et ajoute-les dans Airtable." L'agent chef va : 1. → sous-agent-recherche → 3 leads trouvés 2. → sous-agent-redaction × 3 → 3 emails personnalisés 3. → sous-agent-actions → ajout Airtable 4. → Résumé Slack avec tout ce qui a été fait Si erreur → retry automatique → alerte si échec
