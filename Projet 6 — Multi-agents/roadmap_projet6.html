01
🧠
Comprendre l'architecture multi-agents
agent chef + sous-agents spécialisés
CONCEPT CLÉ
▶

Dans un système multi-agents, l'agent chef reçoit la tâche globale et décide quels sous-agents appeler. Chaque sous-agent est spécialisé dans un domaine précis — il fait une seule chose mais la fait très bien. C'est comme une équipe humaine : un manager qui délègue à des experts.

Dans N8N, les sous-agents sont d'autres workflows appelés via le node Call n8n Workflow Tool. C'est le nouveau concept clé de ce projet !

02
⚙️
Créer l'agent chef (orchestrateur)
AI Agent + system prompt d'orchestration
CONNU ✓
▶

L'agent chef utilise le même node AI Agent que le projet 5. La différence c'est le system prompt — au lieu d'utiliser des outils directs (Gmail, Notion...), il appelle des sous-agents via le node Call n8n Workflow.

System prompt agent chef : "Tu es un orchestrateur IA. Tu reçois des demandes et tu délègues à tes sous-agents spécialisés. - Tâche de recherche → appelle sous-agent-recherche - Tâche de rédaction → appelle sous-agent-redaction - Tâche d'analyse → appelle sous-agent-analyse - Action technique → appelle sous-agent-actions"
03
🔍
Sous-agent 1 — Recherche & leads
Apify + enrichissement + scoring
CONNU ✓
▶

Ce sous-agent reprend la logique du projet 1 — il sait chercher des leads, les enrichir et les scorer. Quand l'agent chef lui donne une cible ("trouve des agences marketing à Paris"), il exécute le pipeline complet et retourne les résultats.

04
✍️
Sous-agent 2 — Rédaction & contenu
GPT spécialisé en copywriting
NOUVEAU
▶

Un agent spécialisé uniquement dans la rédaction — emails de prospection, posts LinkedIn, résumés, propositions commerciales. Son system prompt est optimisé pour le copywriting. L'agent chef lui passe les infos et il retourne le texte rédigé.

Input : { "type": "email_prospection", "entreprise": "Agence X", "contexte": "..." } Output : { "sujet": "...", "corps": "...", "cta": "..." }
05
🔗
Connecter les sous-agents à l'agent chef
Call n8n Workflow Tool — le node clé
NOUVEAU !
▶

Le node Call n8n Workflow Tool permet à l'agent chef d'appeler n'importe quel autre workflow N8N comme un outil. C'est le vrai nouveau concept de ce projet — transformer des workflows en outils pour un agent !

// Dans l'agent chef, chaque sous-agent devient un tool : Tool : "sous-agent-recherche" → appelle le workflow "Sous-agent Recherche Leads" → passe les paramètres → reçoit les résultats

Tester le système complet
